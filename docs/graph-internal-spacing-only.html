Spacing Inside One Graph Widget

This page ignores the outer right VBox. It explains only what happens inside one graph CustomWidget after the VBox has already placed it on screen.

The graph title is not a separate native label. The title, the "Last = ..." text, the header border, the chart, the axes, and the timestamps are all painted inside one CustomWidget.

Relevant Code

const ( chartHeaderHeight = 18 headerChartGap = 2 ) func combinedChartHeight(chartHeight int) int { return chartHeaderHeight + headerChartGap + chartHeight } func splitChartBounds(rect walk.Rectangle) (...) { headerHeight := 18 header = rect top 18px chartY := rect.Y + headerHeight + headerChartGap chart = remaining area } func plotBounds(r walk.Rectangle) walk.Rectangle { return inset(r, 66, 10, 14, 24) }
Graph widget height = 18px header + 2px gap + chart height.
Plot starts 10px below the chartRect top.
Plot starts 66px to the right of chartRect left.
Plot ends 24px above chartRect bottom.
X-axis timestamps are drawn in that 24px bottom inset.
If you measure from the bottom of plotted data to the next graph header, you may include this graph's internal 24px bottom inset plus the outer VBox spacer. That is different from measuring the native space between two CustomWidget rectangles.
